# Cursor Regular Checkup & Optimization Tasks

This document defines **recurring Cursor tasks** to keep the codebase clean, performant,
and free of accumulating technical debt in an AI-assisted workflow.

Use this list **weekly** or **per milestone**.

### 1. Architecture & Structure

- [ ] Split combined concerns (data loading, UI state, rendering)
- [ ] Verify Server vs Client Component boundaries
- [ ] Remove unused abstractions or layers

### 2. State & Persistence
- [ ] Verify persistence strategy (debounce, commit-on-end)
- [ ] Check for unnecessary or chatty writes
- [ ] Ensure no race conditions between UI and backend
- [ ] Confirm ownership of state is explicit

### 3. Rendering & Performance
- [ ] Identify unnecessary re-renders
- [ ] Ensure hooks and callbacks are stable where required
- [ ] Verify RAF throttling / event throttling consistency
- [ ] Remove premature memoization
